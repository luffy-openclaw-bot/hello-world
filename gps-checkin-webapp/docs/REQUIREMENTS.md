# 📋 需求文檔

## 項目背景

客戶需要一個 GPS 打卡系統，用於追蹤外勤員工（On Site/走盤）的工作時間和位置。

---

## 👥 用戶角色

### 1. 員工 (Employee)
- 可以登入系統
- 可以進行 GPS 打卡（上班/下班）
- 可以查看個人打卡記錄
- 可選：拍照上傳

### 2. 管理員 (Admin)
- 可以管理用戶（添加/編輯/刪除員工）
- 可以查看 Dashboard（實時員工位置）
- 可以查看報表
- 可以導出數據
- 可以查看作弊日誌

## 📱 頁面列表

### 員工端
- `/login` - 登入頁
- `/checkin` - 打卡頁
- `/profile` - 個人頁面
- `/records` - 打卡記錄

### 管理員端
- `/admin/login` - 管理員登入
- `/admin/dashboard` - Dashboard
- `/admin/users` - 用戶管理
- `/admin/reports` - 報表
- `/admin/fraud` - 作弊日誌

---

## 🔌 API 列表

### 認證
- `POST /api/login.php` - 登入
- `POST /api/logout.php` - 登出

### 打卡
- `POST /api/checkin.php` - 打卡
- `GET /api/records.php` - 獲取記錄

### 用戶管理
- `GET /api/users.php` - 獲取用戶列表
- `POST /api/users.php` - 創建用戶
- `PUT /api/users.php` - 更新用戶
- `DELETE /api/users.php` - 刪除用戶

### Dashboard
- `GET /api/dashboard.php` - Dashboard 數據
- `GET /api/locations.php` - 員工實時位置

### 報表
- `GET /api/reports.php` - 報表數據
- `GET /api/export.php` - 導出 CSV

### 作弊檢測
- `GET /api/fraud.php` - 作弊日誌

## 🎨 UI/UX 要求

### 設計原則
- 簡潔易用
- 移動優先（員工主要用手機）
- 快速加載
- 清晰的信息架構

### 顏色方案
- 主色：藍色 (#0066CC)
- 成功：綠色 (#00CC66)
- 警告：橙色 (#FF9900)
- 錯誤：紅色 (#FF3333)
